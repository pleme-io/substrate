# Adding an Architecture

Lifecycle for creating and deploying a Pangea infrastructure architecture
using substrate's infra patterns.

---

## Overview

An "architecture" is a reusable infrastructure composition: a set of
cloud resources (S3 bucket, DynamoDB table, KMS key, IAM roles, etc.)
that are wired together and tested as a unit.

```
pangea-architectures/           # Reusable compositions
  lib/state_backend.rb          # Architecture definition
  spec/state_backend_spec.rb    # RSpec synthesis tests

consumer-repo/                  # Repo that deploys the architecture
  flake.nix                     # Uses pangea-infra-flake.nix
  config/workspace.yaml         # Generated by Nix (shikumi pattern)
  inspec/                       # Post-apply verification controls
```

---

## Step 1: Define the Architecture

Create the architecture in `pangea-architectures`:

```ruby
# lib/architectures/state_backend.rb
module Architectures
  class StateBackend
    def initialize(workspace:, region: "us-east-1")
      @workspace = workspace
      @region = region
    end

    def synthesize
      resources = []

      # KMS key for encryption
      kms = kms_key("#{@workspace}-state-key")
      resources << kms

      # S3 bucket for state storage
      resources << s3_bucket("#{@workspace}-state", @region,
        kms_key_id: kms[:arn])

      # DynamoDB table for state locking
      resources << dynamodb_table("#{@workspace}-locks",
        kms_key_id: kms[:arn])

      resources
    end

    private

    def kms_key(name)
      Resources.kms_key(name)
    end

    def s3_bucket(name, region, kms_key_id:)
      Resources.s3_bucket(name, region, kms_key_id: kms_key_id)
    end

    def dynamodb_table(name, kms_key_id:)
      Resources.dynamodb_table(name, kms_key_id: kms_key_id)
    end
  end
end
```

### Security requirements

Every architecture must satisfy the constraints in
[docs/security.md](security.md):

- All storage resources: KMS encryption, `prevent_destroy`, versioning (S3)
- IAM: explicit allow-list, no wildcards
- Tags: `ManagedBy`, `Purpose`, `Environment`, `Team` on every resource
- Secrets: referenced by path, never by value

## Step 2: Write RSpec Synthesis Tests

Synthesis tests verify the full resource graph in pure Ruby --
zero cloud cost, instant execution.

```ruby
# spec/architectures/state_backend_spec.rb
require "spec_helper"

RSpec.describe Architectures::StateBackend do
  let(:arch) { described_class.new(workspace: "test", region: "us-east-1") }
  let(:resources) { arch.synthesize }

  describe "resource presence" do
    it "creates a KMS key" do
      expect(resources).to include_resource_of_type(:kms_key)
    end

    it "creates an S3 bucket" do
      expect(resources).to include_resource_of_type(:s3_bucket)
    end

    it "creates a DynamoDB table" do
      expect(resources).to include_resource_of_type(:dynamodb_table)
    end
  end

  describe "encryption wiring" do
    let(:kms) { resources.find_by_type(:kms_key) }

    it "wires KMS key to S3 bucket" do
      bucket = resources.find_by_type(:s3_bucket)
      expect(bucket[:kms_key_id]).to eq(kms[:arn])
    end

    it "wires KMS key to DynamoDB table" do
      table = resources.find_by_type(:dynamodb_table)
      expect(table[:kms_key_id]).to eq(kms[:arn])
    end
  end

  describe "security" do
    it "enables S3 versioning" do
      bucket = resources.find_by_type(:s3_bucket)
      expect(bucket[:versioning]).to eq(true)
    end

    it "blocks public S3 access" do
      bucket = resources.find_by_type(:s3_bucket)
      expect(bucket[:public_access_block].values).to all(eq(true))
    end

    it "sets prevent_destroy on all stateful resources" do
      stateful = resources.select { |r| [:s3_bucket, :dynamodb_table, :kms_key].include?(r[:type]) }
      stateful.each do |r|
        expect(r[:lifecycle][:prevent_destroy]).to eq(true),
          "#{r[:type]} #{r[:name]} missing prevent_destroy"
      end
    end

    it "tags all resources" do
      resources.each do |r|
        %w[ManagedBy Purpose Environment Team].each do |tag|
          expect(r[:tags]).to have_key(tag),
            "#{r[:type]} #{r[:name]} missing tag: #{tag}"
        end
      end
    end
  end
end
```

Run the synthesis tests:

```bash
cd ~/code/github/pleme-io/pangea-architectures
nix run .#test
```

## Step 3: Create Gated Workspace in Consumer Flake

The consumer repo uses `pangea-infra-flake.nix` or `pangea-infra.nix`
to create a workspace with test gating:

```nix
# consumer-repo/flake.nix
{
  inputs = {
    nixpkgs.url = "github:nixos/nixpkgs?ref=nixos-unstable";
    substrate.url = "github:pleme-io/substrate";
    ruby-nix.url = "github:inscapist/ruby-nix";
  };

  outputs = { self, nixpkgs, substrate, ruby-nix, ... }:
    (import "${substrate}/lib/infra/pangea-infra-flake.nix" {
      inherit nixpkgs ruby-nix;
      flake-utils = nixpkgs;  # or separate flake-utils input
      inherit substrate;
    }) {
      inherit self;
      name = "my-infra";
    };
}
```

This produces gated apps:

```bash
nix run .#test       # RSpec synthesis tests (must pass first)
nix run .#validate   # Pangea config validation
nix run .#plan       # Synthesize + diff (gated on test)
nix run .#apply      # Apply changes (gated on test)
nix run .#verify     # InSpec post-apply verification
nix run .#drift      # Detect configuration drift
nix run .#destroy    # Destroy (explicit confirmation required)
```

### Workspace configuration via Nix (shikumi pattern)

Use `pangea-workspace.nix` to generate the workspace YAML from Nix options:

```nix
pangeaWorkspace = import "${substrate}/lib/infra/pangea-workspace.nix" {
  inherit pkgs;
};

workspaceConfig = pangeaWorkspace.mkWorkspaceConfig {
  name = "production";
  region = "us-east-1";
  architecture = "state-backend";
  variables = {
    bucket_prefix = "pleme-prod";
  };
};
```

This generates a YAML file that the Pangea Ruby DSL reads at runtime --
no shell business logic between Nix evaluation and application execution.

## Step 4: Auto-Generate InSpec Controls

For every RSpec synthesis assertion, create a corresponding InSpec control.
See [docs/testing.md](testing.md) for the mirroring pattern.

```ruby
# inspec/controls/state_backend.rb
control "state-backend-kms" do
  impact 1.0
  title "State backend KMS key exists and is enabled"

  describe aws_kms_key(key_id: input("kms_key_id")) do
    it { should exist }
    it { should be_enabled }
  end
end

control "state-backend-s3" do
  impact 1.0
  title "State backend S3 bucket is secure"

  describe aws_s3_bucket(bucket_name: input("bucket_name")) do
    it { should exist }
    it { should have_versioning_enabled }
    it { should have_default_encryption_enabled }
    it { should_not be_public }
  end
end

control "state-backend-dynamodb" do
  impact 1.0
  title "State backend DynamoDB table exists"

  describe aws_dynamodb_table(table_name: input("table_name")) do
    it { should exist }
    its("billing_mode_summary.billing_mode") { should eq "PAY_PER_REQUEST" }
  end
end
```

### Mirroring rule

Every `expect(...)` in RSpec synthesis should have a corresponding
`describe ... do` in InSpec. This ensures that what you synthesize in
pure Ruby is verified against real cloud resources post-apply.

## Step 5: Deploy with Test Gate

The deployment workflow enforces the test gate:

```bash
# 1. Run synthesis tests (zero cost)
nix run .#test
# -> Layer 1 (unit) + Layer 2 (synthesis) pass

# 2. Plan (shows what would change)
nix run .#plan
# -> Generates diff, no changes applied

# 3. Apply (deploys to cloud)
nix run .#apply
# -> Creates/updates real resources

# 4. Verify (post-apply)
nix run .#verify
# -> Layer 3 InSpec controls run against real resources

# 5. (optional) Check for drift later
nix run .#drift
# -> Compares synthesized config with actual cloud state
```

If any step fails, subsequent steps do not execute.

## Checklist

- [ ] Architecture defined in `pangea-architectures` with resource functions
- [ ] All security constraints satisfied (see `docs/security.md`)
- [ ] RSpec unit tests for each resource function (Layer 1)
- [ ] RSpec synthesis tests for full architecture (Layer 2)
- [ ] Consumer flake created with `pangea-infra-flake.nix`
- [ ] Workspace config generated via `pangea-workspace.nix` (shikumi pattern)
- [ ] InSpec controls mirror RSpec synthesis assertions (Layer 3)
- [ ] Test gate verified: `nix run .#test` must pass before `plan`/`apply`
- [ ] Required tags present on all resources
- [ ] `prevent_destroy` on all stateful resources
- [ ] KMS encryption on all storage resources
