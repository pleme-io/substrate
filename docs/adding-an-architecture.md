# Adding an Architecture

Lifecycle for creating and deploying a typed Pangea infrastructure architecture
that composes typed resource functions, enforces security at the type level,
and is tested at every abstraction layer.

---

## Overview

An "architecture" is a typed Ruby function that composes typed resource
functions into a reusable infrastructure composition. Each resource function
(`S3Bucket`, `DynamodbTable`, `IamPolicy`) enforces its own security
invariants. The architecture function composes them and adds cross-resource
invariants (wiring, least-privilege IAM, tag propagation).

```
pangea-architectures/                  # Reusable compositions
  lib/architectures/state_backend.rb   # Typed architecture function
  lib/resources/                       # Typed resource functions
  spec/resources/                      # Layer 1: resource unit tests
  spec/architectures/                  # Layer 2: architecture synthesis tests
  inspec/controls/                     # Layer 3: auto-generated InSpec controls

consumer-repo/                         # Repo that deploys the architecture
  flake.nix                            # Uses pangea-infra-flake.nix (gated)
  config/workspace.yaml                # Generated by Nix (shikumi pattern)
  inspec/                              # Post-apply verification controls
```

## Step 1: Define the Typed Architecture Function

Create the architecture as a typed Ruby class in `pangea-architectures`.
The class includes the relevant traits and composes typed resource functions.

### Ruby Class Structure

```ruby
# lib/architectures/state_backend.rb
module Pangea
  module Architectures
    class StateBackend
      include Synthesizable
      include Composable
      include SecurityEnforced
      include InSpecMirrorable

      # Required configuration keys (validated before synthesis)
      REQUIRED_CONFIG = [:name, :bucket, :lock_table, :kms_key, :tags].freeze

      # Build the architecture: compose typed resource functions
      def self.build(synth, config)
        validate_config!(config)

        # Each resource function enforces its own invariants
        bucket = Pangea::Aws::S3Bucket.build(synth, {
          bucket_name: config[:bucket],
          kms_key_id: config[:kms_key],
          tags: config[:tags],
        })

        table = Pangea::Aws::DynamodbTable.build(synth, {
          name: config[:lock_table],
          hash_key: 'LockID',
          billing_mode: 'PAY_PER_REQUEST',
          kms_key_id: config[:kms_key],
          point_in_time_recovery: true,
          tags: config[:tags],
        })

        # IAM policy with EXPLICIT actions and resource ARNs
        # Never wildcards -- absolute least-privilege
        policy = Pangea::Aws::IamPolicy.build(synth, {
          name: "#{config[:name]}-access",
          statements: [{
            effect: 'Allow',
            actions: ['s3:GetObject', 's3:PutObject', 's3:ListBucket'],
            resources: [bucket.arn, "#{bucket.arn}/*"],
          }, {
            effect: 'Allow',
            actions: ['dynamodb:GetItem', 'dynamodb:PutItem', 'dynamodb:DeleteItem'],
            resources: [table.arn],
          }],
          tags: config[:tags],
        })

        { bucket: bucket, table: table, policy: policy }
      end

      private

      def self.validate_config!(config)
        REQUIRED_CONFIG.each do |key|
          raise Pangea::ValidationError, "Missing required config key: #{key}" unless config.key?(key)
        end
        validate_required_tags!(config[:tags])
      end

      def self.validate_required_tags!(tags)
        %w[ManagedBy Purpose Environment Team].each do |tag|
          raise Pangea::ValidationError, "Missing required tag: #{tag}" unless tags&.key?(tag)
        end
      end
    end
  end
end
```

### Key Design Points

1. **`self.build(synth, config)`** -- class method, not instance method.
   Architectures are functions, not objects with mutable state.
2. **`validate_config!`** is called first -- fast-fail before any synthesis.
3. **Each child resource function enforces its own invariants** -- the
   architecture does not need to re-validate encryption or versioning.
4. **IAM policies use explicit ARNs from resource outputs** -- never
   string interpolation of account IDs or hardcoded ARNs.
5. **The return value is a hash of resources** -- consumers can access
   individual resources for further composition.

### Security Requirements

Every architecture must satisfy the constraints in
[docs/security.md](security.md):

- All storage resources: KMS encryption (REQUIRED field), `prevent_destroy` (default), versioning (default for S3)
- IAM: explicit allow-list, no wildcards -- validated at the type level
- Tags: `ManagedBy`, `Purpose`, `Environment`, `Team` on every resource -- validated at build time
- Secrets: referenced by path, never by value; `sensitive: true` auto-excluded from state
- Network: default-deny security groups, explicit ingress/egress only

## Step 2: Write RSpec Tests at Every Level

### Layer 1: Resource Function Unit Tests

If any new resource functions are needed, write their unit tests first.
See [docs/testing.md](testing.md) Layer 1 for the full pattern.

### Layer 2: Architecture Synthesis Tests

Synthesis tests verify the full resource graph in pure Ruby --
zero cloud cost, instant execution. They test composition, cross-references,
and security invariants across the entire architecture.

```ruby
# spec/architectures/state_backend_spec.rb
require 'spec_helper'

RSpec.describe Pangea::Architectures::StateBackend do
  let(:synth) { Pangea::Synthesizer.new }
  let(:config) do
    {
      name: 'test-state',
      bucket: 'test-state-bucket',
      lock_table: 'test-state-locks',
      kms_key: 'arn:aws:kms:us-east-1:123:key/test',
      tags: {
        'ManagedBy' => 'pangea',
        'Purpose' => 'state-backend',
        'Environment' => 'test',
        'Team' => 'platform',
      },
    }
  end
  let(:result) { described_class.build(synth, config) }

  describe 'resource presence' do
    it 'creates an S3 bucket' do
      result
      expect(synth.resources).to include_resource_of_type('aws_s3_bucket')
    end

    it 'creates a DynamoDB table' do
      result
      expect(synth.resources).to include_resource_of_type('aws_dynamodb_table')
    end

    it 'creates an IAM policy' do
      result
      expect(synth.resources).to include_resource_of_type('aws_iam_policy')
    end
  end

  describe 'cross-reference wiring' do
    it 'wires bucket ARN into IAM policy resources' do
      bucket_arn = result[:bucket].arn
      policy_resources = result[:policy][:policy_document][:Statement]
        .flat_map { |s| Array(s[:Resource]) }
      expect(policy_resources).to include(bucket_arn)
      expect(policy_resources).to include("#{bucket_arn}/*")
    end

    it 'wires table ARN into IAM policy resources' do
      table_arn = result[:table].arn
      policy_resources = result[:policy][:policy_document][:Statement]
        .flat_map { |s| Array(s[:Resource]) }
      expect(policy_resources).to include(table_arn)
    end

    it 'wires KMS key to S3 encryption' do
      enc = result[:bucket][:server_side_encryption_configuration]
      expect(enc[:rule][:apply_server_side_encryption_by_default][:kms_master_key_id])
        .to eq(config[:kms_key])
    end

    it 'wires KMS key to DynamoDB encryption' do
      expect(result[:table][:server_side_encryption][:kms_key_arn])
        .to eq(config[:kms_key])
    end
  end

  describe 'security invariants' do
    it 'has no wildcard actions in IAM policies' do
      synth.resources_of_type('aws_iam_policy').each do |policy|
        policy[:policy_document][:Statement].each do |stmt|
          stmt[:Action].each do |action|
            expect(action).not_to include('*'),
              "Policy '#{policy[:name]}' has wildcard action '#{action}'"
          end
        end
      end
    end

    it 'has no wildcard resources in IAM policies' do
      synth.resources_of_type('aws_iam_policy').each do |policy|
        policy[:policy_document][:Statement].each do |stmt|
          Array(stmt[:Resource]).each do |resource|
            expect(resource).not_to eq('*'),
              "Policy '#{policy[:name]}' has wildcard resource"
          end
        end
      end
    end

    it 'enables versioning on S3' do
      expect(result[:bucket][:versioning][:enabled]).to eq(true)
    end

    it 'blocks all public access on S3' do
      block = result[:bucket][:public_access_block]
      expect(block.values).to all(eq(true))
    end

    it 'sets prevent_destroy on stateful resources' do
      [result[:bucket], result[:table]].each do |r|
        expect(r[:lifecycle][:prevent_destroy]).to eq(true)
      end
    end

    it 'tags all resources with required tags' do
      synth.resources.each do |r|
        %w[ManagedBy Purpose Environment Team].each do |tag|
          expect(r[:tags]).to have_key(tag),
            "#{r[:type]} '#{r[:name]}' missing tag '#{tag}'"
        end
      end
    end
  end

  describe 'validation' do
    it 'raises when required config keys are missing' do
      expect {
        described_class.build(synth, { name: 'test' })
      }.to raise_error(Pangea::ValidationError)
    end

    it 'raises when required tags are incomplete' do
      expect {
        described_class.build(synth, config.merge(tags: { 'ManagedBy' => 'pangea' }))
      }.to raise_error(Pangea::ValidationError, /tag/)
    end
  end
end
```

Run the synthesis tests:

```bash
cd ~/code/github/pleme-io/pangea-architectures
nix run .#test
```

## Step 3: Create Gated Workspace in Consumer Flake

The consumer repo uses `pangea-infra-flake.nix` or `pangea-infra.nix`
to create a workspace with test gating:

```nix
# consumer-repo/flake.nix
{
  inputs = {
    nixpkgs.url = "github:nixos/nixpkgs?ref=nixos-unstable";
    substrate.url = "github:pleme-io/substrate";
    ruby-nix.url = "github:inscapist/ruby-nix";
  };

  outputs = { self, nixpkgs, substrate, ruby-nix, ... }:
    (import "${substrate}/lib/infra/pangea-infra-flake.nix" {
      inherit nixpkgs ruby-nix;
      flake-utils = nixpkgs;  # or separate flake-utils input
      inherit substrate;
    }) {
      inherit self;
      name = "my-infra";
    };
}
```

This produces gated apps:

```bash
nix run .#test       # RSpec synthesis tests (must pass first)
nix run .#validate   # Pangea config validation
nix run .#plan       # Synthesize + diff (gated on test)
nix run .#apply      # Apply changes (gated on test)
nix run .#verify     # InSpec post-apply verification
nix run .#drift      # Detect configuration drift
nix run .#destroy    # Destroy (explicit confirmation required)
```

### Workspace configuration via Nix (shikumi pattern)

Use `pangea-workspace.nix` to generate the workspace YAML from Nix options:

```nix
pangeaWorkspace = import "${substrate}/lib/infra/pangea-workspace.nix" {
  inherit pkgs;
};

workspaceConfig = pangeaWorkspace.mkWorkspaceConfig {
  name = "production";
  region = "us-east-1";
  architecture = "state-backend";
  variables = {
    bucket_prefix = "pleme-prod";
  };
};
```

This generates a YAML file that the Pangea Ruby DSL reads at runtime --
no shell business logic between Nix evaluation and application execution.

## Step 4: Auto-Generate InSpec Controls

For every RSpec synthesis assertion, create a corresponding InSpec control.
The architecture class includes `InSpecMirrorable`, which enables auto-generation.

### Manual Approach (write controls directly)

```ruby
# inspec/controls/state_backend.rb

control 'state-backend-s3-exists' do
  impact 1.0
  title 'State backend S3 bucket exists'
  desc 'Mirrors: expect(synth.resources).to include_resource_of_type(aws_s3_bucket)'

  describe aws_s3_bucket(bucket_name: input('bucket_name')) do
    it { should exist }
    it { should have_versioning_enabled }
    it { should have_default_encryption_enabled }
    it { should_not be_public }
  end
end

control 'state-backend-dynamodb-exists' do
  impact 1.0
  title 'State backend DynamoDB table exists'
  desc 'Mirrors: expect(synth.resources).to include_resource_of_type(aws_dynamodb_table)'

  describe aws_dynamodb_table(table_name: input('table_name')) do
    it { should exist }
    its('billing_mode_summary.billing_mode') { should eq 'PAY_PER_REQUEST' }
    its('sse_description.status') { should eq 'ENABLED' }
  end
end

control 'state-backend-iam-least-privilege' do
  impact 1.0
  title 'State backend IAM policy has no wildcards'
  desc 'Mirrors: expect(action).not_to include(*)'

  describe aws_iam_policy(policy_name: input('policy_name')) do
    it { should exist }
  end

  policy = aws_iam_policy(policy_name: input('policy_name'))
  policy_doc = JSON.parse(URI.decode_www_form_component(policy.document))
  policy_doc['Statement'].each do |stmt|
    Array(stmt['Action']).each do |action|
      describe "IAM action: #{action}" do
        it('should not contain wildcards') { expect(action).not_to include('*') }
      end
    end
  end
end
```

### Auto-Generation Approach (via `InSpecMirrorable`)

```ruby
# Generate InSpec profile from the architecture class:
profile = Pangea::Architectures::StateBackend.to_inspec_profile(synth, config)
profile.write_to('inspec/controls/')
```

This generates one control per RSpec assertion, with the `desc` field
referencing the original RSpec assertion for traceability.

### Mirroring Rule

Every `expect(...)` in RSpec synthesis MUST have a corresponding
`describe ... do` in InSpec. This ensures that what you synthesize in
pure Ruby is verified against real cloud resources post-apply. See
[docs/testing.md](testing.md) for the complete mapping table.

## Step 5: Deploy with Test Gate

The deployment workflow enforces the test gate:

```bash
# 1. Run synthesis tests (zero cost)
nix run .#test
# -> Layer 1 (unit) + Layer 2 (synthesis) pass

# 2. Plan (shows what would change)
nix run .#plan
# -> Generates diff, no changes applied

# 3. Apply (deploys to cloud)
nix run .#apply
# -> Creates/updates real resources

# 4. Verify (post-apply)
nix run .#verify
# -> Layer 3 InSpec controls run against real resources

# 5. (optional) Check for drift later
nix run .#drift
# -> Compares synthesized config with actual cloud state
```

If any step fails, subsequent steps do not execute.

## Checklist

### Architecture Definition
- [ ] Architecture class defined in `pangea-architectures` under `lib/architectures/`
- [ ] Class includes `Synthesizable`, `Composable`, `SecurityEnforced`, `InSpecMirrorable`
- [ ] `self.build(synth, config)` class method implemented
- [ ] `validate_config!` validates all required keys before synthesis
- [ ] All child resource functions are typed Pangea functions

### Security (Non-Negotiable)
- [ ] KMS encryption on all storage resources (REQUIRED field, not optional)
- [ ] `prevent_destroy` on all stateful resources (default `true`)
- [ ] No wildcards in any IAM policy actions or resources
- [ ] IAM policy ARNs derived from resource outputs, not hardcoded
- [ ] Required tags (`ManagedBy`, `Purpose`, `Environment`, `Team`) on all resources
- [ ] `sensitive: true` on all secret-related fields
- [ ] Default-deny security groups with explicit rules only

### Testing (All Three Layers Required)
- [ ] Layer 1: RSpec unit tests for each new resource function
- [ ] Layer 2: RSpec synthesis tests for architecture composition
- [ ] Layer 2: Security invariant assertions (no wildcards, encryption wired, tags present)
- [ ] Layer 2: Cross-reference wiring assertions (ARNs flow correctly)
- [ ] Layer 2: Validation error assertions (missing config raises)
- [ ] Layer 3: InSpec controls mirror every RSpec synthesis assertion
- [ ] InSpec `desc` fields reference the RSpec assertions they mirror

### Deployment
- [ ] Consumer flake created with `pangea-infra-flake.nix` (gated)
- [ ] Workspace config generated via `pangea-workspace.nix` (shikumi pattern)
- [ ] Test gate verified: `nix run .#plan` runs full RSpec before planning
- [ ] `nix run .#verify` runs InSpec controls post-apply
- [ ] `nix run .#deploy` chains test -> apply -> verify
